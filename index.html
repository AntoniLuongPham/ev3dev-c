The library hierarchic structure

I LEVEL

This level contains functions for a local and remote access to the file system of the EV3 brick. The following code will write the value in the file (on the brick) or will send these data in an UDP message to the brick (on the remote computer).

ev3_init();
ev3_write( "/sys/class/leds/ev3:red:right/trigger", "heartbeat" );
ev3_uninit();

For more information about a remote access to the EV3 brick go to "source/ev3/ev3_link/README".

II LEVEL

Basic functions to read and write attributes of LEDs, see "Using LEDs".

int ok;
uint8_t val;
ev3_init();
ok = get_led_brightness( EV3_GREEN_LEFT, &val );
if ( ok ) {
printf( "EV3_GREEN_LEFT is %s\n", ( val ) ? "ON" : "OFF" );
}
ev3_uninit();

Basic functions to read and write attributes of EV3 Ports, see "EV3 Output Port Driver" and "EV3 Input Port Driver".

char s[ 256 ];
ev3_init();
if ( get_output_state( OUTPUT_A, s, sizeof( s ))) {
printf( "OUTPUT_A state = %s\n", s );
}
ev3_uninit();

Basic functions to read and write attributes of Mindstorms Sensors, see "MINDSTORMS Sensor Class".

uint8_t sn;
int val;
ev3_init();
if ( ev3_search_sensor( LEGO_EV3_TOUCH, &sn, 0 )) {
if ( get_sensor_value( 0, sn, &val )) {
printf( "TOUCH sensor value = %d\n", val );
}
}
ev3_uninit();

Basic functions to read and write attributes of Tacho Motors, see "Tacho Motor Class".

uint8_t sn;
ev3_init();
ev3_tacho_init();
if ( ev3_search_tacho( TACHO, &sn, 0 )) {
set_tacho_regulation_mode( sn, "off" );
set_tacho_run_mode( sn, "time" );
set_tacho_stop_mode( sn, "brake" );
set_tacho_duty_cycle_sp( sn, 100 );
set_tacho_time_sp( sn, 5000 );
set_tacho_ramp_up_sp( sn, 2000 );
set_tacho_ramp_down_sp( sn, 2000 );
set_tacho_run( sn, true );
}
ev3_uninit();

Basic functions to read and write attributes of DC Motors, see "DC Motor Class".

uint8_t sn;
ev3_init();
set_output_mode_inx( OUTPUT_D, OUTPUT_RCX_MOTOR );
ev3_dc_init();
if ( ev3_search_dc_plugged_in( OUTPUT_D, INPUT_MUX__NONE_, &sn, 0 )) {
set_dc_duty_cycle( sn, 100 );
set_dc_command( sn, "run" );
Sleep( 5000 );
set_dc_command( sn, "brake" );
}
ev3_uninit();

Basic functions to read and write attributes of NXT Analog Host, see "NXT Analog Host Driver".

Basic functions to read and write attributes of Multiplexer Input Ports, see "Multiplexer Input Port Driver".

uint8_t sn;
int val;
ev3_init();
set_input_mux_mode_inx( INPUT_1, INPUT_MUX_ANALOG );
set_nxt_analog_host_set_sensor( INPUT_1, INPUT_MUX_1, ( char* ) ev3_sensor_type( LEGO_NXT_TOUCH ));
ev3_sensor_init();
if ( ev3_search_sensor_plugged_in( INPUT_1, INPUT_MUX_1, &sn, 0 )) {
if ( get_sensor_value( 0, sn, &val )) {
printf( "TOUCH sensor value = %d\n", val );
}
}
ev3_uninit();

Basic functions to read and write attributes of Servo Motors, see "Servo Motor Class".

uint8_t sn;
ev3_init();
set_input_mode_inx( INPUT_1, INPUT_NXT_ANALOG );
set_nxt_analog_host_set_sensor( INPUT_1, INPUT_MUX__NONE_, ( char* ) ev3_sensor_type( MS_8CH_SERVO ));
ev3_servo_init();
if ( ev3_search_servo_plugged_in( INPUT_1, INPUT_MUX__NONE_, 1, &sn, 0 )) {
set_servo_command( sn, "run" );
set_servo_position( sn, 100 );
}
ev3_uninit();

III LEVEL

Task-focused user-friendly functions. Only light module in this category at the moment.

ev3_init();
set_light_blink( LIT_LEFT, LIT_AMBER, 1000, 500 );
ev3_uninit();


Python, Ruby, Perl and other languages

Libraries on Python and Ruby have been generated using SWIG. The SWIG interface file called "ev3.i", its source is "ev3.yu-i".

Features of Python library:

  1. Functions return a value as the second element of the result tuple;
  2. Access to descriptors of detected devices only through ev3_sensor_desc, ev3_tacho_desc and ev3_<class>_desc_<field> functions;
  3. Access to other global variables through "ev3" namespace.
1 color = [ '?', 'BLACK', 'BLUE', 'GREEN', 'YELLOW', 'RED', 'WHITE', 'BROWN' ]
2 
3 if not EV3_BRICK:
4  ev3.brick_addr = '192.168.0.244'
5 ev3_init()
6 ev3_sensor_init()
7 
8 ok, sn = ev3_search_sensor( EV3_UART_29 )
9 if ok:
10  set_sensor_mode( sn, 'COL-COLOR' )
11 
12  ok, val = get_sensor_value( 0, sn )
13  if ok:
14  print 'COLOR =', color[ val ]
15 
16 ev3_uninit()

Features of Ruby library:

  1. Functions return a value as the second element of the result tuple, the first element has a boolean type;
  2. Access to descriptors of detected devices only through ev3_sensor_desc, ev3_tacho_desc and ev3_<class>_desc_<field> functions.
1 include Ev3
2 ev3_init()
3 ev3_sensor_init()
4 
5 ok, sn = ev3_search_sensor( LEGO_EV3_TOUCH )
6 if ok
7  while get_sensor_value( 0, sn )[ 1 ] == 0 do end
8 end
9 
10 ev3_uninit()

Features of Perl library:

  1. Functions return a value as the second element of the result tuple;
  2. Access to descriptors of detected devices only through ev3_<class>_desc_<field> functions.
use ev3;
ev3::ev3_init();
ev3::ev3_tacho_init();
my ( $ok, $sn ) = ev3::ev3_search_tacho( $ev3::MINITACHO );
if ( $ok ) {
ev3::set_tacho_regulation_mode( $sn, "off" );
ev3::set_tacho_duty_cycle_sp( $sn, 50 );
ev3::set_tacho_run( $sn, 1 );
sleep( 2 );
ev3::set_tacho_run( $sn, 0 );
}
ev3::ev3_uninit();

Similarly, you can use SWIG to wrap the library in a number of other languages.
